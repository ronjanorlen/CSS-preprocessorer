SASS

Arbetet med Sass och CSS-preprocessorer

Att börja jobba med Sass och CSS-preprocessorer har varit en intresant och lärorik upplevelse. Det kändes inte jättekomplicerat att komma igång men något som jag tyckte var krångligt var att reda ut i vilken partial som ska innehålla vilka delar. Jag misstänker att det kanske blir mer tydligt när det finns mer innehåll på en webbplats och det är flera sektioner och element att hålla reda på.

Som vanligt när ett nytt moment introduceras stöter man alltid på olika typer av problem och det jag har kämpat främst med var att min live server kraschade när jag började lägga in bilder på webbplatsen. Det verkade som att min local host fortsatte köra processer i bakgrunden och cachen uppdaterades inte vilket gjorde att när jag försökte starta min live server på nytt fick jag felmeddelande att den vanliga 1234-porten inte kunde användas samt att mitt projekt inte kunde byggas.

Även om felsökning och hinder längs vägen kan vara frustrerande ibland tycker jag att det är både roligt och intressant att felsöka och det är alltid väldigt lärorikt i slutändan.

Jag kan definitivt se fördelarna med att använda CSS-preprocessorer då det effektiviserar arbetsprocessen avsevärt med att kunna skapa variabler, mixins, färgscheman, partials etc. Det är ett bra sätt att skapa struktur i sin kod och gör det enkelt att ändra, underhålla och felsöka.

Nackdelarna skulle jag säga är om man inte håller bra struktur från början så kommer projektet bli allt svårare med tiden att underhålla och bygga vidare på, eftersom en rörig kodstruktur leder till förvirring och det kan ta väldigt mycket tid att leta rätt på den sektion man vill göra ändringar i eller felsöka.

Sammanfattningsvis har det varit en rolig uppgift att jobba med och jag tror att ju mer bekväm man blir med CSS-preprocessorer desto mer livade webbplatser kommer vi kunna bygga. Jag får någon typ av frihetskänsla av att kunna bli ännu mer kreativ som utvecklare.
